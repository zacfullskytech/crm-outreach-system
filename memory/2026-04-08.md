### Context
Building a lightweight CRM ("Field Notes CRM") for managing client contacts, segmentation, email campaigns, and geographic prospecting targeting veterinary clinics and private medical practices.

- Stack: Next.js 15 App Router + Prisma + Supabase Postgres
- ~1,000 contacts, ~1,000 emails/week target

### What happened today

#### Azure App Service deployment — FAILED (gave up)
- App builds fine locally, production build passes
- GitHub Actions workflow deploys to Azure App Service (Linux, Node 22)
- **Problem**: Azure's Oryx runtime layer extracts `node_modules.tar.gz` to `/node_modules` and symlinks, which interferes with the `.next` build output
- Error: `Could not find a production build in the '.next' directory`
- Tried multiple fixes:
  - `force-dynamic` on all pages
  - Custom startup script (`start-azure.mjs`)
  - Direct node invocation instead of npx
  - Including `node_modules` in release artifact
- None worked — Azure's source-deploy/Oryx behavior kept stripping/restructuring the package
- **Decision**: Abandon App Service source deploy, pivot to Azure Container Apps
